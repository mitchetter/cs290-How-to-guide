Introduction


OMDb is an acronym for the Open Movie Database. The OMDb API is a free web service which allows developers to access movie information. The goal of this how-to guide will be to show the user how to get started using the API and then how to leverage the API to enhance the video inventory system that we developed in assignment4-part2. All code will be written in PHP. We will use the API to fetch the category (or genre) and length (or runtime) of the movie where these had be be manually inputted into the web form previously. We will also add some new fields to the database table: rated and imdbID. With imdbID, we can then fetch the poster for the movie and display that to the user as visual reference as it is added to the inventory system. The service offers two useages - one for data requests and another for movie poster requests. By default, a data call to the API for a particular movie will deliver the following information in JSON format (although XML format is also supported):

  • Title
  • Year (Released)
  • Rated
  • Released (Date)
  • Runtime
  • Genre
  • Director
  • Writer
  • Actors
  • Plot
  • Language
  • Country
  • Awards
  • Poster (URL)
  • Metascore
  • imdbRating
  • imdbVotes
  • imdbID
  • Type
  • Response
